Key Features

集成业务系统，被业务系统集成
  • 业务应用复杂，HyperFlow被企业选为业务中台，为业务应用的开发和运行提供统一的流程开发工具和流程运行引擎。通过符合国际标准的WEB技术，实现对分散在不同业务系统中的数据、流程的统一集成、纳管和自动化驱动。
  • HyperFlow的流程节点，既可以是由人来完成的工作任务，也可以是由后台系统来执行的工作任务。从而实现企业运作的全面自动化，提升数字化转型的成效。
集成业务系统，被业务系统集成
  • HyperFlow通过调用企业业务系统的WebAPI，可调用业务系统功能，驱动业务系统运作，获取业务系统数据。实现把对企业ERP，HRM，SCM，CRM等业务系统的集成。
  • 对业务系统的集成可以采用同步方式，也可以在业务系统完成一个操作所需时间较长时采用异步集成方式，两种集成方式，都可以根据业务系统反馈的数据，来决定流程下一步的运行路径。
  • 反过来，业务系统可以反向集成HyperFlow，例如通过调用HyperFlow的API，启动新流程的运行，完成一个流程节点任务，向流程注入数据等。
通过流程自动化进行业务优化
  • 提供对流程运行数据的智能分析，企业可以直观地对不同流程的运行效率进行分析，指导业务优化；可以通过对阻碍流程效率的节点的分析，识别是人的原因，还是业务系统的原因，从而可以开展针对性的业务优化。
  • 企业也可以从HyperFlow获取流程运行数据，进行更深入的自主分析。
支持多种开发语言
  • HyperFlow的应用接口采用WebAPI方式，支持所有现代编程语言，软件工程师无论是开发企业后台应用，还是前台Web应用，还是iOS/Android移动APP，抑或微信小程序，都可以使用熟悉的编程语言，只需要调用HyperFlow API，即可实现流程的启动、监控，工作列表的获取，流程数据的集成，任务项的挂起、重置、完成等操作；获取工作项的状态数据、业务数据后，可以注入到WEB应用的表单模板中，实现表单的自由定制，或者注入到移动APP的界面上，实现移动APP页面的定制。
流程自动化工程师，可以使用HyperFlow Designer设计流程模板来设计流程模板。
  • 设计流程节点间流向，指定节点的承担者角色
  • 使用SCRIPT节点，在流程中进行编程，可读取、修改、添加流程上下文数据
  • SCRIPT节点，也可以用于以同步或异步方式调用业务系统接口
  • 根据流程上下文数据，决定流程走向， 支持多轮”撤回“，”退回“。 支持异步回调
  • 支持多级”父-子“流程，子流程数据，注入父流程。例如可实现根据子流程的完成情况，决定父流程的流程走向
  • 通过API，指定流程模板和工作团队，即可启动一个流程实例
